# devops-project
Я собрал свой Java‑проект в JAR‑файл, упаковал его в Docker‑образ и загрузил в Docker Desktop. После этого развернул локальный кластер Kubernetes — он поднялся как один узел прямо на моей машине. В PowerShell я применил deployment.yaml с помощью команды kubectl apply -f, затем проверил состояние через kubectl get deployments и kubectl get pods. Все поды вышли в статус Running, то есть контейнеры реально запустились внутри кластера. Теперь их жизненным циклом управляет Kubernetes: если какой‑то контейнер упадёт, кластер сам его перезапустит, а при изменении числа реплик он автоматически масштабирует приложение.
Пока я не создавал Service, поэтому доступ к приложению есть только внутри кластера, но для отладки этого достаточно — главное, что Kubernetes контролирует контейнеры, а не я вручную через docker run.
